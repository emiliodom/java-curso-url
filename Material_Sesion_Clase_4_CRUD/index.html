Sesión 4 — Empezamos el CRUD (con pantallas y base de datos)

Ya corrimos nuestra primera app y entendimos el endpoint. Ahora damos el salto a un CRUD didáctico: guardamos datos en una base persistente (H2 en archivo), y construimos pantallas con Thymeleaf + Bootstrap para que el resultado se vea "real" desde el día 1.

Abrir guía docente Abrir ejercicios Abrir cheat sheet
Meta de la sesión (resultado visible)
Un CRUD mínimo de Libros (o entidad equivalente): listado + formulario + editar + eliminar, y evidencia en H2 Console.

Stack (sin dolor)

  • DB: H2 en archivo (persistente, sin instalar nada).
  • Persistencia: Spring Data JPA (Repository).
  • Vistas: Thymeleaf (templates HTML en el backend).
  • UI: Bootstrap 5 (pulido inmediato).
Reto avanzado: en el proyecto final pueden cambiar Bootstrap por Tailwind si quieren (pero implica más setup).

Checklist (antes de iniciar)

  • Java 17 funciona: java -version
  • Proyecto Spring Boot abre sin errores en el IDE
  • Comprendes el flujo: Request → Controller → Service → Repository → DB
  • Sabes leer el log cuando algo falla
