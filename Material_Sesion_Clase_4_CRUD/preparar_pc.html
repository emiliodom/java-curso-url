Preparar proyecto

Checklist técnico — H2 persistente + Thymeleaf + Bootstrap

Aquí dejamos el proyecto listo para el CRUD con pantallas y datos guardados en archivo.
Idea clave
H2 puede funcionar “en memoria” (se borra al apagar) o “en archivo” (se conserva). Usaremos archivo para que el CRUD se sienta real sin instalar una base externa.

1) Verifica Java 17 y entra a tu proyecto

java -version
javac -version

# Entra a la carpeta del proyecto Spring Boot
# (donde está mvnw.cmd y pom.xml)
.\mvnw.cmd -v
Si falla mvnw.cmd
  • Asegúrate de estar en la carpeta correcta (debe existir mvnw.cmd).
  • Si descargaste un ZIP: descomprime completamente (no ejecutes desde “vista previa”).
  • Si hay bloqueo por antivirus/SmartScreen: reintenta y revisa permisos.

2) Dependencias (Spring Initializr)

Si vas a generar proyecto nuevo, agrega estas dependencias:

  • Spring Web (controladores y rutas)
  • Thymeleaf (vistas HTML)
  • Spring Data JPA (repositorios)
  • H2 Database (base de datos liviana)
Si olvidaste una dependencia (rápido)
Dos opciones:
  • Regenerar en Spring Initializr con las mismas opciones y copiar tu src/ al nuevo proyecto.
  • Editar pom.xml y agregar el starter faltante dentro de <dependencies>.
Ejemplo (H2):
<dependency>
  <groupId>com.h2database</groupId>
  <artifactId>h2</artifactId>
  <scope>runtime</scope>
</dependency>
Luego ejecuta en PowerShell (dentro del proyecto) para descargar:
.\mvnw.cmd -DskipTests package
Decisión didáctica
H2 “archivo” nos da persistencia sin complicaciones. Más adelante podemos migrar a MySQL/PostgreSQL.

3) Configura H2 en modo archivo (persistente)

En src/main/resources/application.properties:

# --- H2 (archivo) ---
spring.datasource.url=jdbc:h2:file:./data/semana4db
spring.datasource.driverClassName=org.h2.Driver
spring.datasource.username=sa
spring.datasource.password=

# JPA (para CRUD inicial)
spring.jpa.hibernate.ddl-auto=update
spring.jpa.show-sql=true

# Consola web de H2
spring.h2.console.enabled=true
spring.h2.console.path=/h2-console
Cómo entrar a H2 Console
  1. Arranca la app.
  2. Abre: http://localhost:8080/h2-console
  3. JDBC URL: jdbc:h2:file:./data/semana4db
  4. User: sa · Password: (vacía)
Error típico
Si el JDBC URL en la consola no coincide exactamente con el de application.properties, parecerá que “no hay tablas”.

4) Estructura recomendada (para no perderse)

src/main/java/.../
  controller/
  service/
  repository/
  model/

src/main/resources/
  templates/
  static/
    css/
Traducción didáctica
Controller = entrada; Service = reglas; Repository = DB; Model = datos; templates = vistas HTML.

5) UI pulida en 2 minutos (Bootstrap en Thymeleaf)

Dentro de tus templates Thymeleaf, agrega Bootstrap por CDN:

<link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.3/dist/css/bootstrap.min.css" rel="stylesheet">

Y opcional: un CSS propio en src/main/resources/static/css/app.css:

<link rel="stylesheet" th:href="@{/css/app.css}" />
Tip: Bootstrap resuelve el 80% del diseño (espaciado, tablas, formularios) sin pelear con CSS desde cero.
