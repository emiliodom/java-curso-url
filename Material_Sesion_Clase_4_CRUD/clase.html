Sesión de Clase 4 — CRUD inicial (H2 archivo + Thymeleaf + Bootstrap)

Unidad 4: reforzar estructura Spring Boot y empezar CRUD con pantallas y persistencia.
Abrir ejercicios

Objetivos (hoy)

  • Reforzar la estructura de un proyecto Spring Boot (carpetas y responsabilidades).
  • Configurar H2 persistente (modo archivo) y usar H2 Console como herramienta.
  • Entender el flujo CRUD: Vista → Controller → Service → Repository → DB.
  • Construir un CRUD mínimo con pantallas: listar, crear, editar, eliminar.

Normas rápidas (para bajar la carga)

  • Primero funciona, luego bonito (Bootstrap ayuda a lo bonito sin distraer).
  • Cuando falle: leer el log y ubicar el punto exacto (template, bean, DB, puerto).
  • Un solo dominio hoy: Recurso didáctico (evita mezclar 2 entidades en la misma sesión).
  • H2 en archivo: si cierran y abren, los datos siguen (eso motiva).

Caso recomendado (para docencia TIC)

“Biblioteca de recursos didácticos”
  • Datos: título, tipo (guía/actividad/video), enlace, nivel/grado, disponible.
  • Pantallas: listado + formulario.
  • Acciones: crear, editar, eliminar.

Guion sugerido (120–140 min)

  • Recordatorio: la sesión anterior logró /hello.
  • Pregunta guía: “¿Qué le falta a una app real?” → datos + pantallas + operaciones.
  • Presenta el flujo del día:
Vista (HTML) -> Controller -> Service -> Repository -> H2 (archivo)

Mapa mínimo (sin teoría extra)
Controller = rutas · Service = reglas · Repository = DB · Model = datos · templates = pantallas
src/main/java/.../
  controller/
  service/
  repository/
  model/

src/main/resources/
  templates/
  static/

  • Explica H2 memoria vs archivo (1 minuto).
  • Configura application.properties y enseña H2 Console.
  • Recalca: JDBC URL debe coincidir.
Resultado esperado
El grupo abre /h2-console y puede ver el esquema/tablas cuando existan.

Haz una demo “paso a paso” y detente en 3 puntos: entity, repository, templates.

  1. Entity DidacticResource (id, title, type, url, available).
  2. Repository DidacticResourceRepository (extends JpaRepository).
  3. Controller MVC (no REST): devuelve vistas Thymeleaf.
  4. Templates: listado + formulario (Bootstrap).
  5. Prueba Create + Read (luego Update/Delete si da tiempo).
Mensaje clave
Un CRUD es una coreografía: si falta 1 parte (template, ruta, repo o DB), se cae. El log te dice cuál.

  • Equipos de 3–4: replican el CRUD del ejemplo.
  • Objetivo mínimo: listado + crear (Read/Create) con H2 persistente.
  • Objetivo extra: editar + eliminar (Update/Delete).
  • Registrar evidencias en el entregable.
Evaluación rápida
¿Corre? ¿se ve? ¿guarda? ¿se puede explicar?

Docente: muestra cómo “leer el error” con calma.

  • Puerto 8080 ocupado → cambiar puerto o matar proceso.
  • Template no encontrado → nombre/ruta en templates/.
  • DataSource fail → revisa URL H2, dependencias y properties.
  • Tabla no aparece → endpoint no se ejecutó / ddl-auto / consola con JDBC distinto.
# Ver procesos usando 8080 (Windows)
netstat -ano | findstr :8080

# Matar proceso
taskkill /PID 12345 /F
