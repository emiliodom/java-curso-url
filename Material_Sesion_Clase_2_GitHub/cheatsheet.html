Cheat Sheet

Git y GitHub — lo mínimo que debes recordar

Referencia rápida para la Sesión 2.

Flujo básico (local)

git status

git add .

git commit -m "Mensaje claro"

git log --oneline
Idea: status te dice la verdad; úsalo seguido.

Publicar en GitHub (remoto)

git branch -M main

git remote add origin https://github.com/USUARIO/REPO.git

git push -u origin main

Mensajes de commit (plantilla)

  • Verbo + objeto: “Agrega README…”, “Corrige texto…”, “Mejora estilos…”
  • Un commit = un propósito (no mezclar 5 cambios distintos).
  • Evita: “cambios”, “final”, “asdf”.

.gitignore mínimo

# OS
.DS_Store
Thumbs.db

# VS Code
.vscode/

# Logs
*.log

Problemas comunes

“No puedo hacer push”
Revisa que estás autenticado en GitHub y que el remoto es correcto.
“Tengo conflicto”
Lee con calma los marcadores, decide qué queda, guarda y vuelve a commitear.

Comandos útiles (extra)

git diff

git restore --staged ARCHIVO

git checkout -b nueva-rama

Cómo evaluar con commits (enfoque docente TIC)

Criterios formativos para evaluar proceso, no solo producto.
  • Historia entendible: el historial “cuenta” qué se hizo y por qué.
  • Commits pequeños: un cambio por commit (sin mezclar temas).
  • Mensajes claros: verbo + objeto (“Agrega…”, “Corrige…”, “Mejora…”).
  • Evidencia de iteración: se ve mejora progresiva (no todo al final).
  • Documentación útil: README permite usar el material sin ayuda.
