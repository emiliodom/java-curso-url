Ejercicio 1 — Crear Controlador base

package com.tugrupo.tuapp.controlador;
@Controller
@RequestMapping("/recursos")
public class RecursoDidacticoControlador {
  private final RecursoDidacticoRepositorio repo;

  public RecursoDidacticoControlador(RecursoDidacticoRepositorio repo) {
    this.repo = repo;
  }
}
Explica por que usamos constructor en lugar de new.

Ejercicio 2 — Ruta GET para listado

@GetMapping
public String list(Model model) {
  model.addAttribute("recursos", repo.findAll());
  return "recursos/lista";
}
Explica que devuelve repo.findAll() y por que se usa en el listado.
Si falla
Revisa que exista templates/recursos/lista.html.

Ejercicio 3 — Ruta GET para formulario

@GetMapping("/nuevo")
public String form(Model model) {
  model.addAttribute("recurso", new RecursoDidactico());

}
Explica que hace model.addAttribute.

Ejercicio 4 — Ruta POST para guardar

@PostMapping
public String create(@ModelAttribute RecursoDidactico recurso) {
  if (recurso.getDisponible() == null) {
    recurso.setDisponible(false);
  }
  repo.save(recurso);
  return "redirect:/recursos";
}
Explica por que se usa redirect:/recursos en lugar de devolver una vista directa.
Por que validar null
Los checkboxes no enviados llegan como null. Si no lo manejas, tu BD puede guardar valores inesperados.

Ejercicio 5 — Formulario conectado (Thymeleaf)

Asegura que el formulario usa th:object y th:field:

<form th:action="@{/recursos}"
      th:object="${recurso}"
      method="post">
  <input type="text" th:field="*{titulo}" required />
  <input type="text" th:field="*{tipo}" required />
  <input type="url" th:field="*{url}" required />
  <input type="checkbox" th:field="*{disponible}" />
  <button type="submit">Guardar</button>
</form>

Crea 3 registros y verifica que aparecen en el listado.
