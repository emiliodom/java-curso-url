Rutas y metodos

Objetivos (hoy)

  • Entender rutas GET/POST y para que sirve cada una.
  • Explicar imports y dependencias con lenguaje docente.
  • Conectar un formulario con @ModelAttribute y guardar.
  • Aplicar un metodo de diagnostico cuando falle una ruta o vista.

Normas rapidas

  • Primero claridad del flujo, luego detalles esteticos.
  • Todo error se entiende leyendo el log con calma.
  • Una ruta = una accion clara (no mezclar tareas).

Caso recomendado

Continuar con "Recursos didacticos".
  • Rutas: listar, formulario, guardar.
  • Campos: titulo, tipo, url, disponible.
  • Objetivo didactico: que el estudiante explique el flujo con ejemplos.

Guion sugerido (120–140 min)

  • Pregunta guia: "Que pasa cuando escribes /recursos en el navegador?"
  • Mapa rapido en la pizarra: URL -> Controlador -> Vista.
  • Meta del dia: crear rutas claras y saber explicarlas.

Mapa minimo
Vista (HTML) -> Controlador -> Repositorio -> BD

Pide que el estudiante diga una frase por capa.

  • Explicar que un import habilita una herramienta concreta.
  • Ejemplo: @GetMapping = ruta que muestra una pagina.
  • Dependencias: Web, Thymeleaf, Data JPA, H2.
Mensaje clave
Si falta una dependencia, no es "tu codigo": es una pieza que el proyecto necesita.

  1. Crear Controlador con @RequestMapping("/recursos").
  2. Agregar @GetMapping para listar.
  3. Agregar @GetMapping("/nuevo") para formulario.
  4. Agregar @PostMapping para guardar.
  5. Probar flujo completo y corregir rutas si fallan.
Evidencia esperada
Formulario crea un registro y redirige al listado.

  • Equipos replican el flujo con su propia entidad.
  • Checklist: rutas correctas + vista carga + POST guarda.
  • Registrar evidencias en el entregable.
No saltarse el paso de explicar
Cada equipo debe explicar el flujo con sus palabras, no solo mostrar la pantalla.

Simula un error y resuelvelo en 4 pasos:

  1. Leer el mensaje real del log.
  2. Ubicar el archivo mencionado.
  3. Revisar ruta y nombre de plantilla.
  4. Reintentar con calma.
