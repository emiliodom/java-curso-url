Checklist tecnico — rutas, plantillas y dependencias

Deja la base lista para trabajar rutas GET/POST y formularios.
Idea clave
Si las dependencias o las carpetas base estan incompletas, las rutas pueden existir pero no cargar vistas. Esta lista evita errores 404 y TemplateInputException.
pom.xml
resources/templates/
java/.../controlador/

1) Verifica Java y Maven Wrapper

java -version
javac -version

# Dentro del proyecto Spring Boot
./mvnw -v
Si falla mvnw
  • Confirma que estas en la carpeta con pom.xml.
  • Si el proyecto vino en ZIP, descomprime completo.
  • En Windows, usa mvnw.cmd.

2) Dependencias minimas

Revisa tu pom.xml y confirma que existen estos starters:

  • spring-boot-starter-web (rutas y controladores)
  • spring-boot-starter-thymeleaf (vistas HTML)
  • spring-boot-starter-data-jpa (repositorios)
  • h2 (base de datos de practica)
Como agregar una dependencia
Ejemplo para H2:
<dependency>
  <groupId>com.h2database</groupId>
  <artifactId>h2</artifactId>
  <scope>runtime</scope>
</dependency>
Luego ejecuta:
./mvnw -DskipTests package

3) Estructura minima de carpetas

src/main/java/.../
  controlador/
  repositorio/
  modelo/

src/main/resources/
  templates/
    recursos/
  static/
    css/
Traduccion
Controlador = entrada, Modelo = datos, Repositorio = BD, templates = plantillas.

4) Prueba rapida de rutas

Crea un Controlador de prueba para verificar que la app responde.

@Controller
public class ControladorPing {
  @GetMapping("/ping")
  @ResponseBody
  public String ping() {
    return "ok";
  }
}
Luego abre http://localhost:8080/ping. Si responde, tus rutas funcionan.

5) Plantillas base (templates)

Crea dos vistas vacias para evitar errores al probar:

templates/recursos/lista.html
templates/recursos/formulario.html
Error tipico
Si el Controlador devuelve "recursos/lista" pero no existe el archivo, aparece TemplateInputException.
