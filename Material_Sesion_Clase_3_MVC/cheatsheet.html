Cheat Sheet

Spring Boot — primer arranque (y MVC mínimo)

Referencia rápida para entorno, comandos, estructura y troubleshooting.

MVC mínimo (para no perderte)

  • Vista: muestra información y captura datos del usuario.
  • Controlador: recibe la petición y decide qué hacer.
  • Modelo: representa datos y reglas del negocio.

Comandos esenciales (Windows)

java -version
javac -version

# Dentro de la carpeta del proyecto
.\mvnw.cmd spring-boot:run

# Probar
http://localhost:8080/
http://localhost:8080/hello
Regla: si el servidor arranca y el endpoint responde, el objetivo está logrado.

MVC → Spring Boot (mapa mental)

Concepto En Spring Boot suele verse como… Ejemplo mental
Controlador @Controller / @RestController Recibe “/voluntarios” y decide respuesta
Modelo Entity / DTO / clases de dominio Voluntario (nombre, teléfono, área…)
Reglas Service Validar, transformar, coordinar pasos
Persistencia Repository Guardar / leer desde base de datos
Vista Plantillas (Thymeleaf) o Frontend separado Formulario y listado
Ojo
MVC es un patrón. Spring Boot es un framework. El framework ayuda a implementar el patrón con estructura.

Errores típicos (y soluciones rápidas)

  • “java no se reconoce” → revisar instalación de JDK 17 y variable PATH.
  • Puerto 8080 ocupado → cerrar la otra app o cambiar el puerto (en sesiones siguientes se verá con más detalle).
  • No responde /hello → revisar que el endpoint tenga @GetMapping("/hello").
  • Imports faltantes → dejar que el IDE autoimporte o buscar “import … GetMapping/RestController”.
