Checklist técnico (Windows)

Preparar la PC para Spring Boot

Objetivo: que java -version funcione, poder generar un proyecto en Spring Initializr y ejecutarlo con mvnw.cmd.

Requisitos mínimos (antes de empezar)

  • Windows 10/11 + permisos para instalar software (ideal: admin).
  • Internet estable (la primera vez Maven descarga dependencias).
  • Java 17 (JDK 17) instalado (Temurin recomendado).
  • Editor/IDE: VS Code / IntelliJ / Eclipse (cualquiera funciona).
  • Espacio libre: ~1–2 GB (dependencias + cachés) y al menos 4 GB RAM (ideal 8 GB).
  • Un descompresor ZIP (Windows lo trae integrado).
Si ves este error:
java : El término 'java' no se reconoce...
Significa: Java no está instalado, o Windows no lo encuentra en el PATH.

Paso 1 — Instalar Java 17 (JDK)

  1. Descarga un JDK 17 (recomendado: Eclipse Temurin 17).
  2. En Windows, instala con el instalador (.msi o .exe).
  3. Recuerda: debe ser JDK (no solo JRE).
Tip
Si no sabes cuál elegir, usa Windows x64 + JDK 17 LTS.

Paso 2 — Verificar si Windows ya encuentra Java

Abre PowerShell nuevo (importante: una ventana nueva) y ejecuta:
java -version
javac -version
Resultado esperado
Que aparezca versión 17 en ambos comandos.

Paso 3 — Configurar PATH (si no funciona)

Si java no se reconoce, hay que agregar el ejecutable a PATH. Lo más común es que Java esté en una ruta parecida a estas: 
C:\Program Files\Eclipse Adoptium\jdk-17\bin
C:\Program Files\Java\jdk-17\bin
Opción A (recomendada): Variables de entorno
  1. Buscar: "Editar las variables de entorno del sistema".
  2. En Variables de usuario o Variables del sistema, editar Path.
  3. Agregar la carpeta ...\bin del JDK 17.
  4. Aceptar todo y abrir una terminal nueva.
Opción B: JAVA_HOME (útil para herramientas)
  1. Crear variable JAVA_HOME apuntando al JDK (sin \bin).
  2. En Path, agregar: %JAVA_HOME%\bin.
Verificación rápida (PowerShell)
Estos comandos ayudan a diagnosticar:
where.exe java
where.exe javac
Muy importante
Si cambiaste variables de entorno, debes cerrar y volver a abrir PowerShell/VS Code para que tome los cambios.

Errores frecuentes (y qué hacer)

Problema Causa típica Solución rápida
“java no se reconoce” JDK no instalado o PATH no configurado Instalar JDK 17 y agregar ...\bin al PATH. Abrir PowerShell nuevo.
Sale Java 8/11 Hay otra Java antes en el PATH where.exe java y ajustar el PATH para priorizar JDK 17.
Se instaló, pero VS Code no lo detecta VS Code estaba abierto antes del cambio Cerrar VS Code y abrirlo de nuevo (recarga variables de entorno).
Descargas lentas / se queda “bajando” Primera descarga de dependencias Esperar 5–15 min. Reintentar con internet estable.
Errores de conexión (proxy/SSL) Red institucional bloquea Maven Central Probar otra red (hotspot). Si no, usar Plan B (Codespaces).
Puerto 8080 ocupado Otra app usa el puerto Cerrar la otra app o cambiar puerto (se ve en sesiones siguientes).
Diagnóstico rápido (PowerShell) 
where.exe java
where.exe javac
java -version
javac -version

Paso 4 — Spring Initializr (paso a paso detallado)

Esto funciona como un “generador de proyecto” para Spring (similar a create-react-app).
  1. Abrir: https://start.spring.io/
  2. En Project:
    • Selecciona Maven.
  3. En Language:
    • Selecciona Java.
  4. En Spring Boot:
    • Deja la última versión estable sugerida.
  5. En Project Metadata (no te preocupes demasiado hoy):
    • Group: algo como com.url o com.equipo.
    • Artifact: nombre corto, sin espacios (ej. demo).
    • Name: puede ser igual al Artifact.
    • Package name: se autogenera; si queda raro, déjalo simple.
  6. En Packaging:
    • Usa Jar.
  7. En Java:
    • Selecciona 17.
  8. En Dependencies (muy importante):
    • Agregar Spring Web.
  9. Click en GENERATE (descarga un ZIP).
  10. Descomprime el ZIP en una carpeta corta (ej: C:\dev\demo).
  11. Abre la carpeta del proyecto en tu editor/IDE.
¿Y si olvidé agregar una dependencia?
Tienes 2 caminos válidos (elige 1):
  • Regenerar en Spring Initializr: vuelve a start.spring.io, agrega la dependencia faltante, descarga y copia tu src/ al proyecto nuevo.
  • Editar pom.xml: agrega el starter dentro de <dependencies> y vuelve a ejecutar .\mvnw.cmd para que descargue.
Ejemplo: si te faltó Spring Web, el starter típico es:
<dependency>
  <groupId>org.springframework.boot</groupId>
  <artifactId>spring-boot-starter-web</artifactId>
</dependency>
Luego, en PowerShell (dentro del proyecto):
.\mvnw.cmd -DskipTests package
Qué archivos buscar (para orientarte)
pom.xml
mvnw.cmd
src\main\java\...\*Application.java
Primer run (Windows)
En PowerShell/CMD dentro de la carpeta del proyecto:
.\mvnw.cmd spring-boot:run
Si el build tarda mucho (primera vez)
La primera ejecución descarga dependencias. Es normal que tarde varios minutos dependiendo del internet.

Paso 5 — Endpoint /hello (para comprobar que funciona)

Crea un Controller mínimo, reinicia la app y prueba en el navegador.
@RestController
public class HelloController {
  @GetMapping("/hello")
  public String hello() {
    return "Hola, Spring Boot";
  }
}
Prueba
http://localhost:8080/hello
Plan B (si todo falla en tu PC)
Si la instalación local se bloquea por permisos, proxy, o Java/Path, usa una alternativa para no detener el aprendizaje:
  • GitHub Codespaces: ejecuta el proyecto en la nube desde el navegador (no requiere Java local).
  • Otra red: si el problema es proxy/SSL, prueba hotspot del teléfono para la primera descarga.
  • JDK portable: descargar JDK 17 en formato ZIP, descomprimir y usar PATH temporal en PowerShell.
  • PC de apoyo: un compañero ejecuta el build/run y tú haces el análisis y documentación (evidencias + explicación).
Ejemplo: PATH temporal (PowerShell)
Solo afecta a la ventana actual de PowerShell.
$env:JAVA_HOME = "C:\dev\jdk-17"
$env:Path = "$env:JAVA_HOME\bin;$env:Path"
java -version
