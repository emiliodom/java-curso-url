Sesión de Clase 3 — Primer arranque de Spring Boot (MVC mínimo)

Unidad 3: mapa mental (responsabilidades) + entorno + primer run + primer endpoint.
Modo clase en vivo
Abre el Guion interactivo (temporizador + checklist + tarjetas “Decir/Hacer/Chequear”) y usa esta guía para el detalle.
Abrir ejercicios

Objetivos (hoy)

  • Usar MVC como mapa mental mínimo: datos vs acciones vs entrada (Controller).
  • Verificar entorno: Java 17 instalado y funcionando.
  • Generar un proyecto con Spring Initializr (equivalente a “create-react-app” para Spring).
  • Ejecutar el proyecto por primera vez con .\mvnw.cmd spring-boot:run (PowerShell).
  • Crear y probar un endpoint /hello.

Normas rápidas (para bajar la carga)

  • Si algo falla: leer el log antes de “adivinar”.
  • No instalar Maven manualmente: usar Maven Wrapper (mvnw).
  • Un solo endpoint hoy: /hello (resultado visible).
  • MVC solo como mapa para ubicar el Controller (no teoría extensa).

Caso recomendado (opcional)

“Registro de voluntarios” (se usa como ejemplo mental para MVC mínimo)
  • Pantallas: formulario + listado.
  • Acciones: registrar, listar, ver detalle.
  • Datos: nombre, teléfono, área, fecha de ingreso, activo.

Guion sugerido (120–140 min)

  • Pregunta detonadora: “¿Qué pasa si una persona hace todo?”
  • Trae ejemplos: dirección, secretaría, docente, estudiante.
  • Cierre: en software también se separan responsabilidades.

3 ideas (sin teoría extra)
Controller = la “entrada” (URLs/endpoints). Model = datos/reglas. Vista hoy no es prioridad (por ahora responderemos texto/JSON). 
Request (navegador) -> Controller -> (Model/Service) -> Response
Chequeo rápido
  • ¿Dónde vive una URL como /hello?
  • ¿Dónde vivirían los “datos” del caso voluntarios?

  • Verificar versiones: java -version y javac -version.
  • Si no aparece Java: confirmar que instaló JDK 17 (no solo JRE).
  • Confirmar que el editor/IDE abre carpetas de proyecto.
Resultado esperado
Cada estudiante deja evidencia de su versión de Java funcionando.

  • Abrir start.spring.io.
  • Elegir: Maven · Java · Java 17.
  • Dependencias: Spring Web.
  • Generar ZIP, descomprimir, abrir carpeta del proyecto.
Mensaje clave
Esto es lo más parecido a “create-react-app” en Spring: un generador de proyecto confiable.

Objetivo: primer build y primer run exitoso (resultado visible en el log).

# Windows (PowerShell / CMD) dentro de la carpeta del proyecto
.\mvnw.cmd spring-boot:run

# Cuando inicie:
http://localhost:8080/
Si algo falla
Verifica Java 17, que estés en la carpeta correcta y revisa el puerto 8080 (conflictos). Mira el log.

Crear un controlador mínimo y probarlo en el navegador.

// src/main/java/.../HelloController.java
@RestController
public class HelloController {
  @GetMapping("/hello")
  public String hello() {
    return "Hola, Spring Boot";
  }
}
Prueba
http://localhost:8080/hello debe responder texto.
MVC mínimo (solo para ubicarte)
Ese archivo es el Controller: recibe la request y devuelve una response.
