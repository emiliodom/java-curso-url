Primer arranque de Spring Boot (paso a paso)

Meta: entorno listo + proyecto que corre + endpoint /hello funcionando (MVC mínimo).
Ir al entregable
Cómo usar estos ejercicios
En lugar de muchos ejercicios pequeños, hacemos un solo objetivo técnico: lograr que Spring Boot corra. Cada paso deja una evidencia clara (ideal para evaluación por proceso).
MVC mínimo (solo como mapa mental)
Controller = entrada (URLs) · Model = datos/reglas. Hoy responderemos texto/JSON (sin pantallas).

Ejercicio 1 — Entorno listo (Java 17)

Objetivo: confirmar que tu computadora está lista para Spring Boot.

  1. Abre una terminal (PowerShell o CMD).
  2. Ejecuta:
java -version
javac -version
Qué debería pasar
  • Debe verse versión 17 (o compatible con 17).
  • Si dice “no se reconoce”: revisar instalación de JDK 17 y PATH.
Evidencia
Captura o copia de la salida de ambos comandos.

Ejercicio 2 — Genera el proyecto y ejecútalo (equipo)

Objetivo: primer build y primer run exitoso (sin instalar Maven manualmente).

  1. Abre https://start.spring.io/.
  2. Selecciona: Maven, Java, Java 17.
  3. Agrega dependencia: Spring Web.
  4. Genera el ZIP, descomprímelo y abre la carpeta en tu editor/IDE.
  5. En la terminal (dentro de la carpeta del proyecto), ejecuta:
.\mvnw.cmd spring-boot:run
Qué debería pasar
  • El log debe mostrar algo como “Started …” y puerto 8080.
  • Prueba: http://localhost:8080/ (aunque sea error 404, significa que el server está vivo).
Resultado esperado
La app corre y se ve en el log el arranque exitoso.

Ejercicio 3 — Endpoint /hello + mini MVC

Objetivo: un resultado visible y explicable (request → Controller → response).

  1. Crea un archivo HelloController.java en tu paquete (junto a la clase principal).
  2. Pega el siguiente código, guarda y reinicia la app si es necesario:
@RestController
public class HelloController {
  @GetMapping("/hello")
  public String hello() {
    return "Hola, Spring Boot";
  }
}
  1. Abre en el navegador: http://localhost:8080/hello.
  2. Escribe 8–12 líneas respondiendo: ¿por qué este archivo es el Controller (MVC mínimo)?
Si te sale error
  • Revisa que tengas import de @GetMapping y @RestController (tu IDE puede autoimportar).
  • Si el puerto 8080 está ocupado, cierra la otra app o cambia el puerto.
  • Si no arranca, vuelve a java -version y al log.
