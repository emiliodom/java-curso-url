1.  **Activación de presaberes**

*Actividad: Discusión sobre experiencias previas con trabajo colaborativo en proyectos, y la importancia de organizar y documentar el progreso en el desarrollo de software.*

*Tiempo estimado: 15 minutos*

*Recursos: Presentación sobre casos reales de proyectos de software y la importancia del control de versiones.*

*Actividad que realizará el estudiante: Compartir experiencias sobre trabajo en equipo, problemas de versionado manual, y reflexionar sobre la necesidad de herramientas de control de versiones.*

2.  **Retroalimentación**

*Actividad: Preguntas y respuestas para introducir conceptos básicos de Git, ramas, y ciclo de vida del software de manera visual y comprensible.*

*Tiempo estimado: 10 minutos*

*Recursos: Diagrama visual del flujo de ramas en Git, foro de discusión en el aula virtual.*

*Actividad que realizará el estudiante: Formular preguntas sobre diferencias entre archivo único vs múltiples versiones, y sobre cómo las ramas pueden representar diferentes fases de desarrollo.*

3.  **Construcción del conocimiento**

***Actividad 1: Introducción a Learn Git Branching***

*Actividad: Demostración en vivo de Learn Git Branching como herramienta gamificada para aprender Git. Exploración de los primeros 4 niveles con visualización interactiva del árbol de Git.*

*Tiempo estimado: 40 minutos*

*Recursos: Learn Git Branching (https://learngitbranching.js.org/?locale=es_AR), proyector para mostrar el juego en tiempo real.*

*Actividad que realizará el estudiante: Acceder a Learn Git Branching, completar niveles 1-4, comprender comandos git commit, git branch, git checkout mediante visualización interactiva.*

*Evaluación: Observación del progreso en el juego y comprensión de conceptos básicos de Git mediante preguntas dirigidas.*

***Actividad 2: Configuración de Git y GitHub***

*Actividad: Configuración práctica de Git local y creación de cuenta GitHub. Demostración de conexión entre repositorio local y remoto.*

*Tiempo estimado: 35 minutos*

*Recursos: Git instalado, navegador web, VS Code con integración Git.*

*Actividad que realizará el estudiante: Configurar git config (user.name, user.email), crear cuenta GitHub, crear primer repositorio, clonar y hacer primer commit.*

*Evaluación: Verificación de configuración correcta y primer repositorio funcional en GitHub.*

4.  **Aplicación del conocimiento**

***Actividad 1: Creación de Estructura de Ramas del Ciclo de Vida***

*Actividad: Desarrollo práctico del primer repositorio con estructura de ramas que refleje el ciclo de vida del software: main, development, feature, hotfix, release.*

*Tiempo estimado: 40 minutos*

*Recursos: VS Code, Git, GitHub, plantilla de estructura proporcionada.*

*Actividad que realizará el estudiante: Crear ramas estratégicas, realizar commits en diferentes ramas, practicar merge básico, documentar proceso en README.md.*

*Evaluación: Evaluación formativa de la estructura de ramas creada y comprensión del propósito de cada una en el ciclo de vida del software.*

***Actividad 2: Práctica Avanzada con Learn Git Branching***

*Actividad: Completar niveles intermedios (5-8) de Learn Git Branching para comprender conceptos de merge y rebase, aplicando inmediatamente en el repositorio personal.*

*Tiempo estimado: 20 minutos*

*Recursos: Learn Git Branching, repositorio personal en desarrollo.*

*Actividad que realizará el estudiante: Avanzar en el juego hasta completar conceptos de merging, aplicar técnicas aprendidas en su repositorio real, documentar comandos utilizados.*

*Evaluación: Verificación del progreso en el juego y aplicación correcta de conceptos en el repositorio personal.*

5.  **Reflexión sobre mi proceso de aprendizaje**

*Actividad: Reflexión escrita sobre la experiencia de aprender Git de manera gamificada y la comprensión del ciclo de vida del software mediante ramas. Los estudiantes evaluarán cómo Git cambia su perspectiva sobre organización y colaboración.*

*Tiempo estimado: 10 minutos*

*Recursos: Plataforma de aula virtual, plantilla de reflexión sobre control de versiones.*

*Actividad que realizará el estudiante: Redactar reflexión sobre la diferencia entre versionado manual vs Git, la utilidad de Learn Git Branching como herramienta de aprendizaje, y cómo las ramas reflejan fases reales del desarrollo.*

*Evaluación: Evaluación cualitativa basada en la comprensión demostrada del valor del control de versiones y la aplicabilidad en proyectos futuros.*

6.  **Introducción al siguiente tema**

*Actividad: Visualización de videos introductorios sobre flujos de trabajo profesionales con Git y metodologías de desarrollo colaborativo.*

*Tiempo estimado: 5 minutos*

*Recursos: Videos seleccionados sobre Git workflow y trabajo en equipo.*

*Git Workflow en Español: https://youtu.be/HiXLkL42tMU*

*GitHub para equipos: https://youtu.be/gaKdJnha5qI*

*Actividad que realizará el estudiante: Tomar notas sobre metodologías de trabajo en equipo con Git y planificar cómo continuará desarrollando su repositorio con más características y mejor documentación.*

7.  **Actividad de cierre y despedida**

*Actividad: Recapitulación de los puntos clave sobre Git, estructura de ramas del ciclo de vida del software, y establecimiento de metas para continuar el desarrollo del repositorio personal.*

*Tiempo estimado: 5 minutos*

*Recursos: Resumen visual de conceptos Git aprendidos, checklist de tareas pendientes para completar el repositorio.*

*Actividad que realizará el estudiante: Formular preguntas finales, confirmar comprensión de las tareas para la siguiente sesión: completar más niveles de Learn Git Branching, mejorar documentación README.md, y practicar más flujos de merge.*
