1.  **Activación de presaberes**

*Actividad: Discusión sobre la diferencia entre tener un proyecto "solo en mi computadora" versus "compartido con el mundo". Analogía con preparar una clase versus impartirla en el aula.*

*Tiempo estimado: 15 minutos*

*Recursos: Presentación sobre la importancia de deployment en el desarrollo web moderno y sus beneficios educativos.*

*Actividad que realizará el estudiante: Compartir experiencias sobre proyectos desarrollados que solo existían localmente, reflexionar sobre el valor de poder compartir trabajo digitalmente con URLs públicas.*

2.  **Retroalimentación**

*Actividad: Preguntas y respuestas para introducir conceptos de deployment usando analogías educativas familiares.*

*Tiempo estimado: 10 minutos*

*Recursos: Diagramas visuales comparando desarrollo local vs producción, foro de discusión sobre beneficios del deployment.*

*Actividad que realizará el estudiante: Formular preguntas sobre el proceso de llevar aplicaciones desde computadora personal hasta ser accesibles globalmente, entender la analogía con compartir conocimiento educativo.*

3.  **Construcción del conocimiento**

***Actividad 1: Introducción a Vercel y Preparación del Proyecto***

*Actividad: Demostración en vivo de Vercel como plataforma de deployment, mostrando cómo preparar el proyecto React del plan de clase para publicación.*

*Tiempo estimado: 40 minutos*

*Recursos: Vercel dashboard, VS Code, proyecto React del plan de clase, proyector para demostración en tiempo real.*

*Actividad que realizará el estudiante: Verificar funcionamiento local de su plan de clase React, optimizar código básico, crear cuenta en Vercel, explorar la interfaz de la plataforma.*

*Evaluación: Verificación de proyecto React funcionando localmente y cuenta Vercel configurada correctamente.*

***Actividad 2: Configuración de Deployment Automático***

*Actividad: Configuración paso a paso de la conexión entre GitHub y Vercel para deployment automático del plan de clase.*

*Tiempo estimado: 35 minutos*

*Recursos: GitHub repository, Vercel platform, guía de configuración paso a paso.*

*Actividad que realizará el estudiante: Conectar repositorio GitHub con Vercel, configurar settings de deployment, entender el flujo automático Git → Build → Deploy.*

*Evaluación: Verificación de conexión exitosa entre repositorio y Vercel, comprensión del flujo automático.*

4.  **Aplicación del conocimiento**

***Actividad 1: Primer Deployment del Plan de Clase***

*Actividad: Realización del deployment completo del plan de clase React, desde Git push hasta URL pública funcionando.*

*Tiempo estimado: 45 minutos*

*Recursos: Proyecto React preparado, Vercel configurado, terminal para Git commands, monitoreo de build process.*

*Actividad que realizará el estudiante: Realizar push a GitHub para trigger deployment, monitorear proceso de build en Vercel, obtener URL pública, verificar funcionamiento de componentes en producción.*

*Evaluación: Evaluación del deployment exitoso con URL pública accesible y aplicación funcionando correctamente en producción.*

***Actividad 2: Verificación y Presentación de Aplicación Desplegada***

*Actividad: Pruebas de funcionamiento en diferentes dispositivos y presentación de URLs públicas del plan de clase desplegado.*

*Tiempo estimado: 20 minutos*

*Recursos: Dispositivos múltiples para pruebas, URLs públicas generadas, proyector para presentaciones.*

*Actividad que realizará el estudiante: Probar su aplicación en diferentes dispositivos, presentar URL pública a compañeros, explicar proceso de deployment realizado y lecciones aprendidas.*

*Evaluación: Verificación de funcionamiento cross-device y presentación exitosa del deployment completado.*

5.  **Reflexión sobre mi proceso de aprendizaje**

*Actividad: Reflexión escrita sobre la experiencia de deployment y la importancia de compartir proyectos digitalmente. Los estudiantes evalúan cómo el deployment completa el ciclo del desarrollo web.*

*Tiempo estimado: 10 minutos*

*Recursos: Plataforma de aula virtual, plantilla de reflexión sobre deployment y enseñanza digital.*

*Actividad que realizará el estudiante: Redactar reflexión sobre la diferencia entre proyectos locales vs públicos, beneficios educativos del deployment, y cómo aplicarían este conocimiento en su futura enseñanza.*

*Evaluación: Evaluación cualitativa basada en la comprensión del valor del deployment y su aplicabilidad en contextos educativos reales.*

6.  **Introducción al siguiente tema**

*Actividad: Visualización de videos introductorios sobre optimización de aplicaciones web y mejores prácticas para deployment.*

*Tiempo estimado: 5 minutos*

*Recursos: Videos seleccionados sobre optimización web y deployment en producción.*

*Deployment con Vercel: https://youtu.be/2HBIzEx6IZA*

*Optimización React: https://youtu.be/E6jWUPlnPZQ*

*Actividad que realizará el estudiante: Tomar notas sobre optimizaciones avanzadas y planificar mejoras futuras para su aplicación desplegada.*

7.  **Actividad de cierre y despedida**

*Actividad: Recapitulación de los puntos clave sobre deployment, celebración de logro de URLs públicas funcionando, y establecimiento de metas para continuar mejorando aplicaciones desplegadas.*

*Tiempo estimado: 5 minutos*

*Recursos: Lista de URLs públicas creadas por estudiantes, resumen visual del proceso de deployment completado.*

*Actividad que realizará el estudiante: Compartir URL pública final, formular preguntas sobre optimizaciones futuras, y planificar cómo usarán deployment en proyectos educativos futuros.*
