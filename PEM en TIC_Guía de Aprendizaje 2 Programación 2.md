**PARTE INTRODUCTORIA**

***"El control de versiones es memoria del proyecto: permite enseñar, aprender y mejorar con evidencia."***

**Nota pedagógica (perfil del curso):** Esta guía está diseñada para estudiantes que se preparan para ser **profesores de Tecnología (TIC)**. El énfasis está en (1) organizar evidencias del proceso, (2) trabajar colaborativamente y (3) traducir conceptos técnicos a lenguaje didáctico.

| 1) Inicializar repositorio Git dentro de `Material_Sesion_Clase_1_HTML/`|

| 3) Crear/editar README con: propósito + cómo abrir offline + listado   |

**SEGUNDA PARTE: CONSTRUCCIÓN DEL PROYECTO (MICRO-ENTREGABLE EN GITHUB)**

1.  **Descripción del proyecto:** Publicación de un repositorio GitHub
  con el material de la Sesión 1 (HTML) bajo control de versiones,
  evidenciando evolución del trabajo a través de commits.

2.  **Fase del proyecto:** Aplicación práctica (organización del trabajo
  y evidencia verificable).

3.  **Actividades a desarrollar:**

    a.  **Repositorio y estructura:**
      - Repositorio creado en GitHub.
      - Archivos del material presentes (HTML/CSS).
      - README con propósito + instrucciones.

    b.  **Historial (commits):**
      - Mínimo 3 commits significativos.
      - Mensajes claros y específicos.

    c.  **Buenas prácticas:**
      - `.gitignore` (si aplica).
      - Orden del proyecto (nombres, carpetas).

    d.  **Entrega:**
      - Link del repositorio.
      - Captura (o link) del historial de commits.
      - Completar la plantilla de entrega: [Material_Sesion_Clase_2_GitHub/entregable_equipo.html](Material_Sesion_Clase_2_GitHub/entregable_equipo.html)

4.  **Evaluación (rúbrica rápida 100 pts):**

- Correctitud: 40
- Lógica y control de flujo: 25
- Organización (métodos): 20
- Legibilidad y comunicación: 15
