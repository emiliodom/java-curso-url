Sesión de Clase 1 — Fundamentos de Java

Unidad 1: variables, operadores, condicionales, ciclos, métodos y buenas prácticas.
Abrir compilador Java

Objetivos (hoy)

  • Entender estructura mínima de un programa Java.
  • Crear variables y operar con ellas.
  • Usar if/else y un ciclo (for/while).
  • Crear un método simple y llamarlo desde main.
  • Aplicar indentación + comentarios pedagógicos.

Normas rápidas

  • Si no compila: leer el error en voz alta.
  • Un cambio a la vez; ejecutar seguido.
  • Nombre de variables claro: edad, nombre.
  • Indentación siempre.

Guion sugerido (140 min)

  • Pregunta detonadora: “¿Cómo le doy instrucciones a una máquina?”
  • Conecta con: algoritmo = pasos, programa = instrucciones, variable = cajita con etiqueta.
Tip
Pide 3 ejemplos cotidianos de algoritmos (receta, cepillado, cruzar calle).

Pega y ejecuta este “Hola Mundo” en Educative.

public class Main {
  public static void main(String[] args) {
    System.out.println("Hola, mundo");
  }
}
  • class: contenedor.
  • main: punto de entrada.
  • System.out.println: imprimir.

public class Main {
  public static void main(String[] args) {
    String nombre = "Ana";
    int edad = 20;
    boolean esEstudiante = true;

    int anioNacimiento = 2026 - edad;

    System.out.println("Nombre: " + nombre);
    System.out.println("Edad: " + edad);
    System.out.println("¿Estudiante?: " + esEstudiante);
    System.out.println("Año nacimiento aprox: " + anioNacimiento);
  }
}
Preguntas rápidas
  • ¿Qué pasa si cambio int por String?
  • ¿Qué significa + cuando hay texto?

public class Main {
  public static void main(String[] args) {
    int edad = 17;

    if (edad >= 18) {
      System.out.println("Eres mayor de edad");
    } else {
      System.out.println("Aún no eres mayor de edad");
    }
  }
}
Conecta con lógica: condición verdadera/falsa, operadores >=, ==, &&, ||.

public class Main {
  public static void main(String[] args) {
    for (int i = 1; i <= 5; i++) {
      System.out.println("Conteo: " + i);
    }
  }
}
Explicación en una frase
“Inicio → condición → incremento, repite hasta que la condición sea falsa”.

  • Levántense 2 minutos, agua, estiren manos/espalda.
  • Micro-check: “¿Cuál fue el error más común hasta ahora?”
  • Recordatorio: 1 cambio → ejecutar → leer error → corregir.

public class Main {
  public static void main(String[] args) {
    saludar("Ana");
    saludar("Luis");
  }

  static void saludar(String nombre) {
    System.out.println("Hola, " + nombre + "!");
  }
}
Mensaje clave
Un método es “una mini receta” reutilizable. Ayuda a leer y mantener el programa.

  • Hagan el Reto final “Perfil” (variables + if + ciclo + método).
  • Meta: todos logran compilar y entender la salida (no importa que sea simple).
  • Si alguien se traba, usar el cheat sheet y pedir ayuda al par (2 minutos).

Objetivo: aplicar variables + control de flujo + métodos con una tarea que exige explicar, probar y depurar.

  • Formar equipos de 3–4. Roles sugeridos: Driver, Navigator, Tester, Documentador (rotan cada 10 min).
  • Elegir un reto principal (A o B) en ejercicios.html y completar además el Reto C (depuración).
  • Producto: completar la plantilla de entregable en entregable_equipo.html (código + pruebas + explicación corta).
  • Si no alcanzan a terminar, se deja como tarea y se entrega en la siguiente sesión.
Preguntas guía (docente)
¿Cuál es la entrada? ¿Cuál es la salida? ¿Qué condición decide? ¿Cuándo termina el ciclo? ¿Qué se puede convertir en método?

  • Revisión de pares: “¿se entiende qué hace el código?”
  • Ticket de salida: 1 aprendizaje + 1 duda.
